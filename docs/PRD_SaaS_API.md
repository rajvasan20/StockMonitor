# Stock Monitor → SaaS API Product Plan

## Context

Transform the Stock Monitor CLI tool into a **subscription API service** for Registered Investment Advisors (RIAs) and wealth managers. Pre-built report library for NIFTY 100, accessed via API. Two report types at launch: **Management Integrity** and **Industry Overview**. Goldman/Morgan Stanley-style PDF output.

**Target:** B2B API for Indian RIAs/Advisors
**Timeline:** 2-4 weeks (proper MVP)
**Delivery:** API-first, pre-built report library
**Coverage:** NIFTY 100 at launch
**Reports:** Management Integrity + Industry Overview
**Style:** Goldman/MS — narrative, detailed exhibits, executive summary up front

---

## Phase 1: Report Quality (Week 1)

### 1A. Batch Pipeline — Extract All NIFTY 100

**Goal:** Pre-build the foundation layer (MD&A + Notes markdowns) for all 100 companies.

**What exists:**
- `red_flag/downloader.py` — downloads annual reports (4-source fallback)
- `/extract-sections` skill — reads PDF, produces MD&A + Notes context markdowns
- `management_integrity/analyzer.py` — builds integrity reports from MD&A
- `/industry` skill — builds cross-company sector summaries

**What to build:**
- `batch/pipeline.py` — orchestration script that:
  1. Downloads missing annual reports for all NIFTY 100 (5 years each)
  2. Tracks which companies have MD&A context files vs which are missing
  3. Generates integrity reports for companies with complete data
  4. Groups companies by sector, generates industry overviews
  5. Logs progress, handles failures gracefully, supports resume

- `batch/nifty100_registry.py` — master registry of 100 companies with:
  - Ticker, BSE code, Screener slug, sector classification
  - Extend from existing `red_flag/downloader.py` NIFTY50 + EXTRA_TICKERS

**CLI:**
```
python run.py batch --step download      # Step 1: download all PDFs
python run.py batch --step extract       # Step 2: extract MD&A + Notes (via Claude)
python run.py batch --step integrity     # Step 3: generate integrity reports
python run.py batch --step industry      # Step 4: generate industry overviews
python run.py batch --step pdf           # Step 5: convert to Goldman-style PDFs
python run.py batch --step all           # Run full pipeline
python run.py batch --status             # Show progress dashboard
```

### 1B. Goldman/MS-Style PDF Generator

**Goal:** Convert markdown reports to polished, institutional-grade PDFs.

**What to build:** `reports/pdf_generator.py`

**Design elements (Goldman/MS style):**
- **Cover page:** Company name, report type, date, "Confidential" watermark
- **Executive Summary:** 3-5 bullet verdict up front (first thing the reader sees)
- **Table of Contents:** Auto-generated from markdown headers
- **Body:** Clean serif font (Georgia/Garamond), section headers in dark blue
- **Exhibits:** Tables with alternating row shading, clear headers
- **Verdict boxes:** Colored callout boxes for MET/MISSED/EXCEEDED verdicts
- **Footer:** Page numbers, disclaimer, "Generated by [Brand Name]"
- **Back page:** Methodology note, data sources, disclaimer

**Tech:** `weasyprint` or `reportlab` for PDF generation from HTML/CSS template.

**Templates needed:**
- `reports/templates/integrity_report.html` — single-company integrity
- `reports/templates/industry_report.html` — sector overview
- `reports/templates/base.css` — Goldman-style typography and layout

## Phase 2: API Layer (Week 2)

### 2A. FastAPI Application

**What to build:** `api/` module

```
api/
    __init__.py
    app.py              # FastAPI application
    auth.py             # API key validation + rate limiting
    routes/
        reports.py      # Report endpoints
        catalog.py      # Available reports listing
        health.py       # Health check
    models.py           # Pydantic schemas
    rate_limiter.py     # Per-tier rate limiting
```

**Endpoints:**

| Method | Path | Description | Response |
|---|---|---|---|
| GET | `/v1/reports/catalog` | List all available reports | JSON: tickers, sectors, dates |
| GET | `/v1/reports/integrity/{ticker}` | Get integrity report | PDF binary or JSON metadata |
| GET | `/v1/reports/industry/{sector}` | Get industry report | PDF binary or JSON metadata |
| GET | `/v1/reports/integrity/{ticker}/json` | Raw structured data | JSON (for API integrations) |
| GET | `/v1/health` | API health check | JSON |

**Report store:** Local filesystem initially (`output/reports_store/`), migrate to S3 later.

```
output/reports_store/
    integrity/
        ABB_integrity_v2025.pdf
        ABB_integrity_v2025.json      # metadata + structured data
    industry/
        electrical_equipment_v2025.pdf
        electrical_equipment_v2025.json
    catalog.json                       # master index
```

### 2B. Auth + Subscription Tiers

**API Key model:**
- Each client gets a unique API key (UUID)
- Key maps to a subscription tier
- Rate limiting enforced per key

**Tiers:**

| Tier | Reports/Day | Report Types | Price (suggested) |
|---|---|---|---|
| Free | 3 | Catalog + sample reports only | Rs 0 |
| Starter | 15 | Integrity only | Rs 2,999/mo |
| Professional | 50 | Integrity + Industry | Rs 7,999/mo |
| Enterprise | Unlimited | All + raw JSON data | Rs 19,999/mo |

**Storage:** SQLite initially (`data/api/subscriptions.db`), migrate to Postgres later.

**Schema:**
```sql
CREATE TABLE api_keys (
    key TEXT PRIMARY KEY,
    client_name TEXT,
    email TEXT,
    tier TEXT,           -- free/starter/professional/enterprise
    created_at TIMESTAMP,
    expires_at TIMESTAMP,
    is_active BOOLEAN
);

CREATE TABLE usage_log (
    id INTEGER PRIMARY KEY,
    api_key TEXT,
    endpoint TEXT,
    ticker_or_sector TEXT,
    timestamp TIMESTAMP,
    response_status INTEGER
);
```

### 2C. Rate Limiting

- In-memory counter per API key (reset daily at midnight IST)
- Redis later for multi-instance deployment
- Return `429 Too Many Requests` with `X-RateLimit-Remaining` header

## Phase 3: Payment + Onboarding (Week 3)

### 3A. Payment Integration

**Razorpay** (preferred for India B2B):
- Subscription API for recurring billing
- Webhook for payment success → activate API key
- Support UPI, cards, net banking

**Flow:**
1. RIA visits landing page → selects tier → pays via Razorpay
2. Webhook fires → creates API key → emails key to client
3. Client uses API key in `Authorization: Bearer <key>` header

**What to build:**
- `api/routes/onboarding.py` — subscription creation + Razorpay checkout
- `api/webhooks.py` — Razorpay webhook handler
- Landing page (simple HTML) with pricing table + checkout buttons

### 3B. Admin Dashboard

Simple admin panel to:
- View active subscriptions
- Monitor API usage per client
- Manually create/revoke API keys
- Track revenue

**Tech:** Simple Flask/FastAPI admin route with basic HTML, no framework needed.

## Phase 4: Polish + Launch (Week 4)

### 4A. Documentation

- **API docs:** Auto-generated from FastAPI (Swagger/ReDoc at `/docs`)
- **Getting Started guide:** How to get API key, make first request, parse response
- **Sample code:** Python, curl, Postman collection

### 4B. Deployment

**Option A (simple):** Railway / Render
- Single container deployment
- Managed Postgres addon
- Auto-SSL, custom domain
- ~$20-50/month for MVP traffic

**Option B (cost-optimized):** VPS (Hetzner/DigitalOcean)
- $10-20/month for a capable VM
- Docker compose: FastAPI + SQLite
- Caddy for reverse proxy + SSL
- Manual but cheaper

### 4C. Landing Page

Single-page site:
- What it does (3 bullet points)
- Sample report (watermarked PDF preview)
- Pricing tiers
- "Get API Key" CTA → Razorpay checkout
- API docs link

## Key Risks

| Risk | Mitigation |
|---|---|
| PDF extraction quality varies across companies | Batch pipeline tracks quality scores; manual review for top 20 |
| Claude context limits on large annual reports | Already handled — `/extract-sections` works on full PDFs |
| Screener.in rate limiting | Existing 3s+jitter delay; batch runs overnight |
| Low initial demand | Free tier drives trial; sample reports as marketing |
| Report freshness | Weekly batch refresh; version reports with generation date |
